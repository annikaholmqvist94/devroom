# Plan 06: BFF (Backend-for-Frontend)

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implementera BFF som tunn REST-fasad mot frontend. Validerar user-JWT, propagerar JWT i Authorization-header till interna tjänster (Auth, User, Message), returnerar deras svar oförändrade. Vid plan-slut kan klient prata enbart med BFF för signup/login/profil/messages.

**Architecture:** Spring Boot 4 + Spring Web. JWT-validering via `auth-starter`. WebClient (eller RestClient) mot interna tjänster. Inga egen affärslogik — BFF är genuint tunn. Public endpoints: signup/login. Auth-required: messages, profil-uppslag.

**Tech Stack:** Spring Boot 4, Spring Web, Spring Security, `auth-starter`, RestClient.

**Refererar spec:** sektion 2.1, 4.2.

**Pre-condition:** plan 01-05 klara.

## Task 3: JWT-validator + filter + security config

Identiskt mönster som i Message Service (plan 05 task 9). Public endpoints är `/auth/**` (signup + login).

```java
@Configuration
@EnableWebSecurity
public class SecurityConfig {
    @Bean
    public SecurityFilterChain chain(HttpSecurity http, JwtAuthenticationFilter filter) throws Exception {
        http
            .csrf(csrf -> csrf.disable())
            .sessionManagement(s -> s.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(a -> a
                .requestMatchers("/auth/**", "/actuator/**").permitAll()
                .anyRequest().authenticated())
            .addFilterBefore(filter, UsernamePasswordAuthenticationFilter.class);
        return http.build();
    }
}
```

## Task 5: AuthProxyController

```java
@RestController
@RequestMapping("/auth")
public class AuthProxyController {

    private final RestClient authClient;

    public AuthProxyController(RestClient authClient) {
        this.authClient = authClient;
    }

    @PostMapping("/signup")
    public ResponseEntity<Map<String, Object>> signup(@RequestBody Map<String, Object> body) {
        return authClient.post()
                .uri("/auth/signup")
                .contentType(MediaType.APPLICATION_JSON)
                .body(body)
                .retrieve()
                .toEntity(new ParameterizedTypeReference<>() {});
    }

    @PostMapping("/login")
    public ResponseEntity<Map<String, Object>> login(@RequestBody Map<String, Object> body) {
        return authClient.post()
                .uri("/auth/login")
                .contentType(MediaType.APPLICATION_JSON)
                .body(body)
                .retrieve()
                .toEntity(new ParameterizedTypeReference<>() {});
    }
}
```

## Task 6: MessagesProxyController (kräver JWT, propagera den)

```java
@RestController
@RequestMapping("/messages")
public class MessagesProxyController {

    private final RestClient messageClient;

    public MessagesProxyController(RestClient messageClient) {
        this.messageClient = messageClient;
    }

    @PostMapping
    public ResponseEntity<Map<String, Object>> post(
            @RequestBody Map<String, Object> body,
            @RequestHeader("Authorization") String authHeader) {
        return messageClient.post()
                .uri("/messages")
                .header("Authorization", authHeader)
                .contentType(MediaType.APPLICATION_JSON)
                .body(body)
                .retrieve()
                .toEntity(new ParameterizedTypeReference<>() {});
    }

    @GetMapping
    public ResponseEntity<List<Map<String, Object>>> get(
            @RequestParam UUID channelId,
            @RequestParam(required = false) Instant since,
            @RequestHeader("Authorization") String authHeader) {
        return messageClient.get()
                .uri(uri -> {
                    var b = uri.path("/messages").queryParam("channelId", channelId);
                    if (since != null) b.queryParam("since", since.toString());
                    return b.build();
                })
                .header("Authorization", authHeader)
                .retrieve()
                .toEntity(new ParameterizedTypeReference<>() {});
    }
}
```

Commit.

## Task 7: UsersProxyController (för frontend att läsa profil)

För nuvarande räcker det att proxa GET /users/{id} via REST mot User Service. User Service har bara gRPC i plan 03, så vi behöver lägga till en REST-endpoint där.

**Decision-point:** Är det bättre att (a) lägga till REST-endpoint i User Service eller (b) anropa gRPC från BFF?

(a) är simpler. Lägg till en REST-endpoint i User Service som lättviktigt återexponerar samma data:

```java
// services/user-service/src/main/java/com/devroom/user/web/UserController.java
@RestController
@RequestMapping("/users")
public class UserController {
    private final UserRepository repo;
    public UserController(UserRepository repo) { this.repo = repo; }

    @GetMapping("/{id}")
    public ResponseEntity<Map<String, Object>> get(@PathVariable UUID id) {
        return repo.findById(id)
                .map(u -> Map.<String, Object>of(
                        "userId", u.getUserId(),
                        "displayName", u.getDisplayName(),
                        "teamId", u.getTeamId(),
                        "isSystem", u.isSystem(),
                        "mentorPersonality", u.getMentorPersonality()))
                .map(ResponseEntity::ok)
                .orElseGet(() -> ResponseEntity.notFound().build());
    }
}
```

(Lägg till i User Service som en del av plan 6 — eller skuffa ut till en separat micro-task i plan 03 retroaktivt. Bekvämast: lägg till nu i denna plan.)

I BFF:

```java
@RestController
@RequestMapping("/users")
public class UsersProxyController {
    private final RestClient userClient;
    public UsersProxyController(RestClient userClient) { this.userClient = userClient; }

    @GetMapping("/{id}")
    public ResponseEntity<Map<String, Object>> get(@PathVariable UUID id,
                                                     @RequestHeader("Authorization") String auth) {
        return userClient.get()
                .uri("/users/{id}", id)
                .header("Authorization", auth)
                .retrieve()
                .toEntity(new ParameterizedTypeReference<>() {});
    }
}
```

User Service behöver också aktivera JWT-validering (samma filter som Message Service). Detta är en kopiering av plan 05 task 9 in i user-service.

## Task 9: Manuell smoke-test

```bash
docker compose -f docker-compose.dev.yml up -d
# Starta auth, user, message, bff
mvn -pl services/auth-service spring-boot:run &
mvn -pl services/user-service spring-boot:run &
mvn -pl services/message-service spring-boot:run &
mvn -pl services/bff spring-boot:run &
sleep 25

# Hela flödet via BFF (port 8080)
RESP=$(curl -s -X POST http://localhost:8080/auth/signup \
  -H 'Content-Type: application/json' \
  -d '{"email":"bff@test.com","password":"password123"}')
TOKEN=$(echo $RESP | jq -r .jwt)
USER_ID=$(echo $RESP | jq -r .userId)

sleep 3

curl -X POST http://localhost:8080/messages \
  -H "Authorization: Bearer $TOKEN" \
  -H 'Content-Type: application/json' \
  -d '{"channelId":"33333333-3333-3333-3333-333333333301","body":"Hello via BFF"}'

curl "http://localhost:8080/messages?channelId=33333333-3333-3333-3333-333333333301" \
  -H "Authorization: Bearer $TOKEN" | jq

curl "http://localhost:8080/users/$USER_ID" \
  -H "Authorization: Bearer $TOKEN" | jq
```

Expected: alla anrop fungerar end-to-end via BFF.
