**Goal:** Extend the CI workflow to a CD pipeline that builds the 6 Devroom images in GitHub Actions and pushes them to GHCR on push to main, and wire the Helm chart to pull from GHCR.

## Task 1: Add the `images` job to ci.yml

**Files:**
- Modify: `.github/workflows/ci.yml`

- [ ] **Step 1: Append the `images` job under `jobs:`**

In `.github/workflows/ci.yml`, find the end of the existing `helm` job:

```yaml
      - name: Helm template (render check)
        run: helm template devroom helm/devroom > /dev/null
```

Add directly after it (same indentation as the `build:`/`helm:` job keys — 2 spaces):

```yaml

  images:
    # Publish container images to GHCR only for merged code (push to main),
    # not on PRs. PRs still run the build + helm jobs for validation.
    if: github.event_name == 'push' && github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    permissions:
      contents: read
      packages: write
    strategy:
      matrix:
        include:
          - service: auth-service
            dockerfile: services/auth-service/Dockerfile
          - service: user-service
            dockerfile: services/user-service/Dockerfile
          - service: message-service
            dockerfile: services/message-service/Dockerfile
          - service: gateway
            dockerfile: services/gateway/Dockerfile
          - service: bot-service
            dockerfile: services/bot-service/Dockerfile
          - service: frontend
            dockerfile: frontend/Dockerfile
    steps:
      - uses: actions/checkout@v6

      - name: Log in to GHCR
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build and push ${{ matrix.service }}
        uses: docker/build-push-action@v6
        with:
          context: .
          file: ${{ matrix.dockerfile }}
          push: true
          tags: |
            ghcr.io/annikaholmqvist94/${{ matrix.service }}:${{ github.sha }}
            ghcr.io/annikaholmqvist94/${{ matrix.service }}:latest
```

(The image is tagged with the full commit SHA for traceability plus `latest`. The build context is the repo root `.` — matching `deploy.sh` — so each Dockerfile's `COPY services/<svc>/...` / `COPY frontend/...` paths resolve.)

- [ ] **Step 1: Create `docs/adr/0015-cicd-ghcr.md`**

```markdown
# ADR-0015: CI/CD — bygg + publicera images till GHCR

**Status:** Accepted
**Date:** 2026-07-15

## Context

Fas C börjar med CI/CD. Devroom hade CI (mvn verify + helm lint/template) men ingen
CD — images byggdes bara lokalt. Vi vill att pipelinen publicerar container-images.
GitHub Actions-runners kan inte nå den lokala Minikuben, så CI kan inte deploya lokalt.

## Decision

Lägg ett `images`-matrisjobb i `ci.yml` som bygger de 6 Devroom-imagesarna och pushar
till **GHCR** (`ghcr.io/annikaholmqvist94/<tjänst>`) med SHA- + `latest`-taggar, bara på
push till `main`. Auth via inbyggda `GITHUB_TOKEN` (`packages: write`). En
`values-ghcr.yaml` wire:ar chartet att dra images från GHCR. Kluster-deploy hör till
Fas D (EKS, där GitHub kan nå klustret).

## Considered alternatives

### 1. Docker Hub
Kräver externa secrets och ger ingen fördel över GHCR (som är gratis + GITHUB_TOKEN).

### 2. Self-hosted runner på laptopen
Skulle låta GitHub nå Minikube, men mer drift och säkerhetsyta. Avvisat.

### 3. GitOps / ArgoCD (pull-modell)
Löser "GitHub når inte klustret" genom att ArgoCD i klustret drar från git + GHCR.
Kraftfullt men mer komplexitet; uppskjutet som egen framtida plan för att hålla Plan 16
fokuserad.

## Consequences

- PR:er kör `build` + `helm` (test/lint) men publicerar inga images; bara mergad kod
  (push till main) publiceras.
- Chartet får en andra values-fil: lokal (`devroom`/`Never`) vs GHCR
  (`ghcr.io/...`/`IfNotPresent`) — inga mall-ändringar (samma portabilitet som Fas D).
- **dev-mentor** (externt repo) publiceras inte av denna CD; en ren GHCR-deploy måste
  förse dess image separat.
- Verifierbart LIVE på GitHub (till skillnad från Plan 13–15): imagesarna dyker upp under
  repots Packages efter merge till main.
```
